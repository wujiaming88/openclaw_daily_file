# Part 3：双雄对比分析

## 基本信息对比

| 维度 | OpenAI (Ryan Lopopolo) | Anthropic (Prithvi Rajasekaran) |
|------|----------------------|-------------------------------|
| **发布日期** | 2026-02-11 | 2026-03-24 |
| **实验规模** | 5 个月，100 万行代码，真实产品 | 多次实验，从前端到全栈 |
| **核心工具** | Codex (GPT-5) | Claude Agent SDK (Opus/Sonnet 4.5→4.6) |

| **代码量** | ~1M 行 | 单应用级 |

## 核心哲学对比

### OpenAI："人类掌舵，Agent 执行"
- **关注点**：如何让整个工程团队以 Agent-first 方式运作
- **隐喻**：Agent 是新员工 → 需要好的入职培训、清晰的架构、可执行的规范
- **核心主张**：纪律体现在脚手架（scaffolding）而非代码本身

### Anthropic："Generator-Evaluator 对抗式反馈"
- **关注点**：如何让单个 Agent（或少量 Agent）在长会话中保持高质量输出
- **隐喻**：GAN 的对抗训练 → Generator 生成，Evaluator 评判，反馈循环驱动改进
- **核心主张**：分离做事和判断是可靠性的关键杠杆

## 关键差异

### 1. 长会话策略

| 策略 | OpenAI | Anthropic |
|------|--------|-----------|
| 上下文管理 | 渐进式披露，~100 行 AGENTS.md 作为地图 | Context Reset（清空+交接）或 SDK 自动压缩 |
| 上下文焦虑 | 未明确提及 | 核心问题，Sonnet 4.5 严重，Opus 4.5 解决 |
| 持久性 | 仓库即状态，一切版本化 | 结构化交接产物在会话间传递 |

### 2. 质量保证

| 策略 | OpenAI | Anthropic |
|------|--------|-----------|
| 审查方式 | Agent-to-agent review + Ralph Wiggum 循环 | 独立 Evaluator Agent（GAN 式） |
| 自我评估 | 鼓励 Codex 审查自己 + Agent 互审 | 明确发现自评不可靠 → 必须分离 |
| 人工参与 | 可选审查，推向更少 | 设定标准和权重，循环自动化 |
| 机械执行 | 自定义 linter + CI + "黄金原则" | Sprint 合同 + 阈值检查 |

### 3. 规模与场景

| | OpenAI | Anthropic |
|--|--------|-----------|
| **目标** | 百万行产品代码库持续维护 | 单应用从零到完整 |
| **时间尺度** | 持续 5+ 个月 | 单次运行 20 分钟～6 小时 |
| **复杂度** | 多领域、多层架构、CI/CD、运维 | 全栈应用（React+FastAPI） |
| **团队** | 多工程师并行 | 多 Agent 串行 |

### 4. 对待"AI Slop"

| | OpenAI | Anthropic |
|--|--------|-----------|
| **问题** | Agent 复制已有模式（含不好的） → 漂移 | 缺乏干预时 Claude 倾向安全/平庸的设计 |
| **解法** | 黄金原则 + 定期扫描 + 自动修复 PR | 评分标准明确惩罚 AI slop + 评估器反馈循环 |

## 共识与共鸣

两篇文章虽然视角不同，但有强烈共鸣：

### 1. Harness 是真正的工程难点
- OpenAI："纪律体现在脚手架而非代码"
- Anthropic："harness design 是 agentic coding 前沿的关键"

### 2. 约束 > 指令
- OpenAI：机械执行的架构约束是乘数
- Anthropic：Sprint 合同+硬阈值 > 泛泛的提示

### 3. 环境决定上限
- OpenAI："早期慢是因为环境欠规范"
- Anthropic："naive 实现必然失败——两个失败模式都是环境问题"

### 4. 分离关注点
- OpenAI：人类掌舵 vs Agent 执行
- Anthropic：Generator vs Evaluator

### 5. 迭代而非完美
- OpenAI：修正便宜，等待昂贵
- Anthropic：5-15 轮迭代，允许失败和转向

## 互补视角
