# 中国软考高级资格·系统架构设计师（SA）备考全攻略

> 研究员：黄山 (wairesearch) · 日期：2026-03-17

## 执行摘要

系统架构设计师（SA, System Architect）是中国计算机技术与软件专业技术资格（水平）考试（简称"软考"）的**高级资格**科目。通过后等同于**高级工程师（副高）职称**，无需评审直接认定。本报告整理了最新的考试政策、大纲、题型、教材、备考策略和资源，供系统备考参考。

---

## 1. 考试概况

### 1.1 基本信息

| 项目 | 说明 |
|------|------|
| **全称** | 系统架构设计师 (System Architect) |
| **级别** | 高级资格 |
| **对应职称** | 高级工程师（副高） |
| **考试形式** | 全国统一机考（2023年下半年起改为上机考试） |
| **考试频次** | **一年两次**（2024年起调整，上半年5月 + 下半年11月） |
| **报考条件** | **无限制**——不限学历、年龄、工作经验，可直接报考高级 |
| **合格标准** | 每科满分75分，各科均达**45分**即通过 |
| **证书有效期** | 永久有效（2024年起新政策取消了有效期限制） |
| **官方网站** | https://www.ruankao.org.cn/ |

| 批次 | 考试时间 | 报名时间（预估） |
|------|----------|------------------|

## 2. 考试科目与题型

### 2.1 三科总览

系统架构设计师考试分为**3个科目**，需要**同一次考试中三科全部通过**（单科成绩不保留）：

| 科目 | 题型 | 题量 | 分值 | 时长 |
|------|------|------|------|------|
| **科目一：综合知识** | 选择题（单选） | 75题 | 75分 | 150分钟（最短120分钟可交卷） |
| **科目二：案例分析** | 问答题 | 5题选答（一般必做1+选做1-2） | 75分 | 与论文连考，总计210分钟 |
| **科目三：论文** | 写作题 | 4选1 | 75分 | 与案例连考，总计210分钟 |

### 2.2 机考改革后的时间安排（2023年下半年起）

```
上午  8:30 — (10:30~11:00)    综合知识（选择题，150分钟）
      ↓ 连考
上午  (10:30~11:00) — 12:30   案例分析（问答题）
      ├── 案例分析最短作答 60 分钟
      ├── 案例分析最长作答 90 分钟  
      └── 案例+论文总时长 210 分钟

下午  14:30 — 16:30            论文（写作题，120分钟）
```

> **重要变化**：综合知识和案例分析在上午**连考**，考生做完综合知识后可直接进入案例分析（综合知识最少做120分钟后才可提前交卷进入案例）。论文下午单独考。

### 2.3 各科目详解

#### 科目一：综合知识（选择题）

**75道单选题，每题1分**，覆盖面极广：

| 知识领域 | 预估占比 | 重点内容 |
|----------|----------|----------|
| **软件架构设计** | ~20% | 架构风格（管道-过滤器、MVC、微服务等）、ABSD、DSSA、ATAM评估 |
| **软件工程** | ~15% | 需求工程、设计模式（23种）、UML建模、测试方法 |
| **计算机网络** | ~10% | OSI/TCP、网络安全、VPN、防火墙 |
| **信息安全** | ~10% | 加密算法（对称/非对称）、数字签名、PKI、访问控制 |
| **数据库** | ~8% | 关系数据库设计、规范化、NoSQL、数据仓库 |
| **操作系统** | ~5% | 进程管理、内存管理、文件系统 |
| **新技术** | ~10% | 云计算、大数据、AI、物联网、区块链、边缘计算、数字孪生 |
| **数学与算法** | ~5% | 图论、概率统计、组合、线性规划 |
| **法律法规/标准化** | ~5% | 知识产权、著作权、专利、标准化机构 |
| **专业英语** | ~5% | 英文阅读理解（约5题） |
| **项目管理等其他** | ~7% | 项目管理基础、系统工程等 |

**备考策略**：历年真题覆盖率约60-70%，刷真题是性价比最高的备考方式。

#### 科目二：案例分析（问答题）

**5道大题，一般1-2题必做 + 若干选做题**：

| 高频考点 | 说明 |
|----------|------|
| **架构设计与评估** | ATAM方法、质量属性（可用性、性能、安全性、可修改性）、架构风格选择 |
| **设计模式** | 23种GoF设计模式的适用场景和UML类图 |
| **数据库设计** | ER图、关系模式、规范化、Redis缓存设计 |
| **Web架构** | 分层架构、微服务、RESTful API、负载均衡 |
| **云原生架构** | 容器化、Kubernetes、服务网格、DevOps |
| **安全架构** | WPDRRC模型、网络安全体系、数据库安全 |
| **大数据架构** | Hadoop、Spark、Lambda/Kappa架构 |
| **UML建模** | 用例图、类图、序列图、状态图的读图和填空 |

**题型特点**：
- 通常包含 **图表填空**（UML图、架构图补全）
- 需要**分析论述**（为什么选择某种架构/模式，优缺点对比）
- 2024年起新增了更多**实际案例场景**分析

#### 科目三：论文（写作题）

**4选1，写一篇2000-3000字的论文**：

| 论文结构 | 要求 |
|----------|------|
| **摘要** | 300-330字，概述项目背景和个人角色 |
| **正文** | 2000-2500字，论述技术方案和实践经验 |
| **总字数** | 不少于2500字（含摘要） |

**近年论文真题方向**（2020-2025）：

| 年份 | 论文方向 |
|------|----------|
| 2025上 | 微服务架构设计、系统可靠性设计 |
| 2024下 | 云原生架构设计、面向服务架构 |
| 2024上 | 大数据架构、软件架构评估 |
| 2023下 | 系统集成、软件测试策略 |
| 2023上 | 云原生、数据分片技术 |

**论文三大核心**：
1. **格式规范** — 摘要+正文结构清晰，字数达标
2. **内容充实** — 需要结合**真实项目经验**（或合理虚构），体现架构师水平
3. **写作速度** — 2小时写2500+字，机考打字速度是硬性要求

## 4. 通过率与难度分析

### 4.1 通过率数据

| 来源 | 通过率 |
|------|--------|
| 希赛网数据 | **5%-8%**（官方公布口径） |
| CSDN/知乎经验帖 | **10%-15%**（考虑弃考后的实际到考通过率） |
| 某地2025年公布数据 | 整体软考约14%通过 |

> 通过率看似很低，但主要原因是：
> 1. **弃考率高** — 很多人报名后没有认真复习
> 2. **三科单独划线** — 任何一科没过就算失败
> 3. **论文拉分** — 很多技术强但写作弱的考生倒在论文上

### 4.2 难度分析

| 科目 | 难度 | 说明 |
|------|------|------|
| 综合知识 | ⭐⭐⭐ | 面广但不深，刷真题可有效提分 |
| 案例分析 | ⭐⭐⭐⭐ | 需要理解+表达能力，不能只靠记忆 |
| 论文 | ⭐⭐⭐⭐⭐ | 最大难点，需要项目经验+写作功底+速度 |

## 5. 备考策略

### 5.1 总体时间规划（建议3-6个月）

```
┌─────────────────────────────────────────────────────────────────┐
│ 第一阶段（1.5-2个月）：基础夯实                                    │
│ · 通读教材重点章节（架构、安全、数据库、新技术）                      │
│ · 配合视频课程理解知识点                                           │
│ · 每天 1-2 小时                                                   │
├─────────────────────────────────────────────────────────────────┤
│ 第二阶段（1-1.5个月）：强化突破                                    │
│ · 刷历年真题（2019-2025），按科目分类练习                           │
│ · 重点攻克设计模式（23种）、架构风格、质量属性                      │
│ · 开始准备论文素材（2-3个项目经验）                                 │
│ · 每天 2-3 小时                                                   │
├─────────────────────────────────────────────────────────────────┤
│ 第三阶段（2-4周）：冲刺模考                                       │
│ · 完整模拟考试（按真实时间做套题）                                  │
│ · 论文限时练习（2小时写完2500字）                                  │
│ · 错题回顾，查漏补缺                                              │
│ · 考前几页纸/三色笔记                                             │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 各科目备考策略

#### 综合知识（目标：50-55分）

**核心策略：刷真题 + 记高频考点**

1. **历年真题至少做5套**（2020-2025年），标注错题
2. **高频考点优先**：
   - 架构风格（必考3-5题）
   - 设计模式（必考2-3题）
   - 信息安全（必考5-8题）
   - 数据库设计（必考3-5题）
   - UML图（必考2-3题）
3. **英语题不放弃**：5分不算少，技术类英语有规律可循
4. **新技术关注**：云原生、AI、大数据每年必考

#### 案例分析（目标：50-55分）

**核心策略：理解原理 + 背模板 + 练表达**

1. **必背内容**：
   - ATAM 架构评估方法（效用树、场景优先级）
   - 质量属性6大类（可用性、性能、安全性、可修改性、易用性、可测试性）
   - 23种设计模式的意图和适用场景
   - 数据库规范化（1NF→BCNF）
2. **答题模板**：
   - 架构风格题：定义 → 优点 → 缺点 → 适用场景
   - 设计模式题：模式名称 → 意图 → 结构（类图） → 适用性
   - 质量属性题：属性定义 → 战术/策略 → 权衡分析
3. **练习图表填空**：UML类图、序列图、组件图的补全

#### 论文（目标：45-50分）

**核心策略：准备素材 + 练模板 + 练速度**

1. **准备2-3个项目素材**：
   ```
   项目名称：XXX管理系统
   我的角色：系统架构师
   时间：202X年X月-X月
   背景：XX行业，用户量XXX，核心挑战是XXX
   架构方案：采用了XXX架构/技术
   效果：上线后XXX指标提升了XX%
   ```

2. **论文万能模板**：
   ```
   【摘要】300字
   项目背景（什么项目、什么行业）→ 我的角色 → 
   采用了什么技术/方法 → 取得了什么效果

   【正文】
   第一段（400字）：项目背景详述
   第二段（600字）：技术方案论述（核心！展示架构师水平）
   第三段（600字）：实施过程和遇到的问题及解决方案
   第四段（400字）：效果总结和个人反思
   ```

3. **打字速度训练**：
   - 机考要求2小时写2500+字
   - 建议考前至少**完整模拟3次**
   - 目标打字速度：**40字/分钟以上**

4. **高频论文方向准备**（2026年预测）：
   - ⭐⭐⭐⭐⭐ 微服务架构设计
   - ⭐⭐⭐⭐⭐ 云原生架构
   - ⭐⭐⭐⭐ 系统可靠性/高可用设计
   - ⭐⭐⭐⭐ 大数据架构
   - ⭐⭐⭐ 软件架构评估（ATAM）
   - ⭐⭐⭐ 安全架构设计
   - ⭐⭐⭐ 面向服务架构（SOA）

## 7. 高频知识点速查

### 7.1 架构风格（必考）

| 架构风格 | 特征 | 适用场景 |
|----------|------|----------|
| **管道-过滤器** | 数据流驱动，各环节独立 | 编译器、数据处理流水线 |
| **分层架构** | 严格分层，上层依赖下层 | 企业应用、操作系统 |
| **客户端-服务器** | C/S 模式 | 传统桌面应用 |
| **MVC** | 模型-视图-控制器分离 | Web 应用 |
| **微内核** | 核心小+插件扩展 | IDE、浏览器 |
| **微服务** | 服务独立部署、轻量通信 | 大型互联网应用 |
| **事件驱动** | 发布-订阅、事件总线 | 消息系统、GUI |
| **SOA** | 服务注册、WSDL、ESB | 企业集成 |

### 7.2 设计模式分类（必考）

| 类型 | 模式（共23种） |
|------|----------------|
| **创建型**（5种） | 工厂方法、抽象工厂、单例、建造者、原型 |
| **结构型**（7种） | 适配器、桥接、组合、装饰器、外观、享元、代理 |
| **行为型**（11种） | 责任链、命令、解释器、迭代器、中介者、备忘录、观察者、状态、策略、模板方法、访问者 |

### 7.3 质量属性及战术（必考）

| 质量属性 | 常用战术 |
|----------|----------|
| **可用性** | 心跳检测、冗余、故障切换、回滚 |
| **性能** | 缓存、负载均衡、资源池、异步处理 |
| **安全性** | 认证授权、加密、审计日志、入侵检测 |
| **可修改性** | 高内聚低耦合、中间件、接口隔离 |
| **可测试性** | 依赖注入、Mock、日志记录 |
| **易用性** | 用户反馈、撤销操作、向导式界面 |

### 7.4 ATAM 架构评估方法（必考）

```
步骤1：介绍ATAM方法
步骤2：介绍业务驱动
步骤3：介绍架构
步骤4：识别架构方法
步骤5：生成质量属性效用树
步骤6：分析架构方法
步骤7：讨论和排序场景
步骤8：分析架构方法（重复步骤6）
步骤9：呈现结果
```

输出：**风险点、非风险点、敏感点、权衡点**

## 8. 机考注意事项

### 8.1 考试环境

- 使用考场提供的电脑，不能自带设备
- 考场发草稿纸，其他物品不能带入
- 机考系统有倒计时，注意时间分配

### 8.2 时间分配建议

| 科目 | 建议用时 | 提醒 |
|------|----------|------|
| 综合知识 | 90-100分钟做题 + 20-30分钟检查 | 不确定的先标记 |
| 案例分析 | 60-90分钟 | 先读题，选自己擅长的 |
| 论文 | 全部120分钟 | 留10分钟审题+构思 |

### 8.3 论文机考特别提醒

- **打字速度决定成败**——2小时要打2500+字
- 建议提前用**搜狗/微软拼音**练习（考场一般提供这两种输入法）
- 先在**草稿纸上列提纲**（5分钟），再开始打字
- 注意**分段清晰**，阅卷老师看得舒服更容易给高分

## 10. 总结与建议

### 给不同背景考生的建议

| 人群 | 建议 |
|------|------|
| **有5年+开发经验** | 优势明显，重点攻论文和新技术，2-3个月足够 |
| **有架构设计经验** | 直接用真实项目写论文，通过率极高 |
| **应届生/经验少** | 需要4-6个月，论文素材要提前编排 |
| **非计算机专业** | 新版大纲弱化编码，突破IT管理+论文写作可行 |

### 核心备考原则

1. **真题为王** — 综合知识靠真题覆盖60-70%考点
2. **论文先行** — 论文是最大风险点，不要最后才准备
3. **三科平衡** — 任何一科不过就全部白费
4. **模拟实战** — 至少完整模考2-3次
5. **打字速度** — 机考时代，速度就是分数
