## 一、架构审查总结

### 1.1 v2.0 方案评价

v2.0 方案整体成熟度 **8/10**，架构分层清晰，"AI 不直接下单"的安全红线正确。核心亮点是 AI 决策层（分钟~日级）和 Trading Engine（毫秒~秒级）的分离。

### 1.2 发现的问题（按优先级）

| 优先级 | 问题 | 建议 |
|-------|------|------|
| 🔴 P0 | API Key 在 WebSocket URL query string 中 | 改用 Authorization header 或首帧认证 |
| 🔴 P0 | 引擎本地熔断必须不依赖 AI 层 | 文档明确：熔断是引擎自主行为，AI 只做事后分析 |
| 🟡 P1 | AI 调参无时间窗口累积限制 | 24h 累计调参幅度上限 50% |
| 🟡 P1 | WebSocket 重连后不同步状态 | 重连后先 GET /risk/status 再恢复监听 |
| 🟡 P1 | 审计日志不可篡改仅依赖 DB 触发器 | 增加链式哈希或 append-only 文件副本 |
| 🟡 P1 | 多 Agent 超时无 fallback | 加 runTimeoutSeconds + 缺失数据默认中性 |
| 🟢 P2 | 综合评分公式过于机械 | LLM 决策优先，评分公式作为护栏 |
| 🟢 P2 | 健康检查脚本无限重启 | 加冷却期或改为只告警 |
| 🟢 P2 | 缺优雅停机设计 | 补 graceful shutdown 章节 |
| 🟢 P2 | Prompt 硬编码在 Python 中 | 改用 YAML/Jinja2 模板文件 |

### 1.3 架构方向性讨论

经过从第一性原理的推导，结论：

- **v2.0 的闭环控制方向是正确的**——以 2026 年 LLM 能力（Claude Opus 4.6）为基础，AI 作为决策大脑是可行的
- **闭环的绝对优势在"人不在时"的值守场景**——美股夜盘、港股午盘、出差期间，AI 持续监控和决策不可替代
- **但 v2.0 缺少一个关键机制：信任的梯度**——权限是静态二值的（能做/不能做），应该是动态渐进的

最终决策：**保持 v2.0 完整架构，增补信任梯度模型**。

## 二、增补方案：信任梯度模型

### 2.1 设计理念

> **以终为始：架构按全自动设计，权限渐进开放。**
>
> AI 不是一开始就被信任，也不是永远被限制。
> AI 用实际表现赢得权限，用真实数据驱动升级。
> 人随时可以一键降级，保留最终控制权。

```
终态目标：
  AI 全自主交易 → 人只做监督和极端干预

渐进路径：
  OBSERVER → GUARDIAN → ADVISOR → OPERATOR → AUTOPILOT
  （只读）   （安全方向）（需确认） （低危自动）（全自动）

时间线：
  第 1 周    第 1 月     第 3 月     第 6 月     人工审计后
```

### 2.2 信任等级定义

```
┌─────────────────────────────────────────────────────────────────┐
│                     信任等级体系（5 级）                          │
│                                                                  │
│  Level 0: OBSERVER（观察者）                                     │
│  ├── AI 权限：只读 + 生成报告                                   │
│  ├── 写操作：全部需要人手动执行                                  │
│  ├── 价值：验证 AI 分析质量，积累决策记录                        │
│  └── 典型交互："AI 说建议暂停策略" → 人自己输入 /futu stop       │
│                                                                  │
│  Level 1: GUARDIAN（守护者）                                     │
│  ├── AI 权限：只读 + 安全方向自动执行                            │
│  ├── 可自动执行：暂停策略、缩小仓位、收窄网格、推送告警          │
│  ├── 不可自动执行：启动策略、加仓、扩大网格、下单                │
│  ├── 核心原则：AI 误判的最坏后果 = 错过上涨（可承受）            │
│  └── 典型交互：凌晨暴跌 → AI 自动暂停策略 → 人醒后决定是否重启  │
│                                                                  │
│  Level 2: ADVISOR（顾问）                                        │
│  ├── AI 权限：双向操作，全部需要人确认                           │
│  ├── AI 可以提议：启停策略、调参、加减仓                          │
│  ├── 确认方式：Telegram 内联按钮 [执行] [拒绝]                   │
│  ├── 超时处理：30 分钟无确认 → 自动取消提议                      │
│  └── 典型交互：AI 推送"建议扩大网格" + [执行] 按钮 → 人点击确认  │
│                                                                  │
│  Level 3: OPERATOR（操作者）                                     │
│  ├── AI 权限：低危操作自动执行，高危操作需确认                   │
│  ├── 自动执行：策略启停、参数微调（幅度 < 15%）                  │
│  ├── 需确认：参数大幅调整（> 15%）、新增策略、大额操作            │
│  ├── 护栏：综合评分公式作为兜底（AI 决策与评分冲突时取保守值）    │
│  └── 典型交互：AI 自动调整网格区间 → 推送通知 → 人事后审阅       │
│                                                                  │
│  Level 4: AUTOPILOT（全自动）                                    │
│  ├── AI 权限：全部操作自动执行                                   │
│  ├── 约束：仍受引擎硬性风控限制（熔断、止损、仓位上限）          │
│  ├── 人的角色：监督仪表盘 + 每周审阅 + 异常干预                  │
│  ├── 解锁条件：必须人工审计后主动解锁（不可自动达到）            │
│  └── 典型交互：AI 全自主运行 → 人看日报/周报 → 满意就不管        │
└─────────────────────────────────────────────────────────────────┘
```

#### Trading Engine AI Bridge 改造

```python
# api/routers/ai_bridge.py — 增加信任等级检查

class TrustGate:
    """信任门控：每个 AI 操作先过信任检查"""
    
    OPERATION_MIN_TRUST = {
        "read_positions":      TrustLevel.OBSERVER,
        "generate_report":     TrustLevel.OBSERVER,
        "push_alert":          TrustLevel.OBSERVER,
        "pause_strategy":      TrustLevel.GUARDIAN,
        "reduce_position":     TrustLevel.GUARDIAN,
        "narrow_grid":         TrustLevel.GUARDIAN,
        "start_strategy":      TrustLevel.ADVISOR,
        "expand_grid":         TrustLevel.ADVISOR,
        "adjust_params_minor": TrustLevel.ADVISOR,
        "adjust_params_major": TrustLevel.OPERATOR,
        "place_order":         TrustLevel.OPERATOR,
        "cancel_all":          TrustLevel.AUTOPILOT,
    }
    
    OPERATION_NEEDS_CONFIRM = {
        # ADVISOR 级别：所有写操作需确认
        TrustLevel.ADVISOR: ["start_strategy", "expand_grid", 
                             "adjust_params_minor", "adjust_params_major",
                             "place_order"],
        # OPERATOR 级别：只有高危操作需确认
        TrustLevel.OPERATOR: ["adjust_params_major", "place_order", 
                              "cancel_all"],
        # AUTOPILOT：只有极高危需确认
        TrustLevel.AUTOPILOT: ["cancel_all"],
    }
    
    async def check(self, operation: str, current_trust: TrustLevel) -> GateResult:
        min_trust = self.OPERATION_MIN_TRUST.get(operation)
        if min_trust is None:
            return GateResult(allowed=False, reason="Unknown operation")
        
        if current_trust.value < min_trust.value:
            return GateResult(
                allowed=False,
                reason=f"Trust level {current_trust.name} < required {min_trust.name}"
            )
        
        needs_confirm = operation in self.OPERATION_NEEDS_CONFIRM.get(current_trust, [])
        return GateResult(allowed=True, needs_confirmation=needs_confirm)
```

## 三、其他增补建议

### 3.1 单节点简化方案（可选）

如果新加坡 VPS 无其他用途，建议将 OpenClaw 迁移到香港 VPS：

- 消除 Tailscale VPN 跨区域依赖
- Trading Engine API 变为 localhost 调用（延迟 0）
- 减少 $12/月 成本
- 减少一个故障点

保留两台 VPS 的理由：新加坡 VPS 还承载 SaaS 平台等其他服务。

### 3.2 综合评分公式的定位调整

v2.0 的综合评分公式不应作为唯一决策路径，改为**护栏**：

```
决策流程：
  1. LLM 输出结构化决策（MAINTAIN/ADJUST/PAUSE/STOP + 具体参数）
  2. 综合评分公式独立计算
  3. 对比：
     - LLM 和评分一致 → 执行
     - LLM 比评分更激进 → 取保守值
     - LLM 比评分更保守 → 按 LLM 执行（保守方向不限制）
     
  核心原则：评分公式是下限，不是决策者
```

### 3.3 审计日志加固

```
链式哈希方案：
  每条审计日志的 checksum = SHA256(本条内容 + 上一条 checksum)
  篡改任何一条 → 后续所有 checksum 校验失败
  
  + append-only 文本文件副本（/opt/futu-trading/logs/ai_audit.jsonl）
  + 文件权限 0644，属主 root（Trading Engine 用户无法修改）
```

## 四、结论

v2.0 方案的架构方向正确，以终为始的闭环 AI 控制是最终目标。增补信任梯度模型后：

1. **架构从第一天就支持全自动**（以终为始）
2. **权限从 OBSERVER 开始**（安全起步）
3. **AI 用实际表现赢得信任**（数据驱动，不拍脑袋）
4. **任何时候一键降级**（人的最终控制权）
5. **AUTOPILOT 必须人工审计解锁**（不可能意外触发）
