# Part 2：Active Memory —— 主动记忆

## 核心问题

> "大多数记忆系统能力强但被动。它们依赖主 Agent 决定何时搜索记忆，或依赖用户说'记住这个'或'搜索记忆'。到那时，记忆本应让回复自然流畅的时刻已经过去了。"

传统记忆 = **文件柜 + 糟糕的 UX**。系统技术上知道怎么搜索记忆，但只在用户以正确方式、正确时机、正确措辞请求时才会搜。

## 什么是 Active Memory

Active Memory 是一个**可选的、阻塞式的记忆子 Agent**，在主回复生成**之前**运行，主动搜索并注入相关记忆。

### 运行时流程

```
用户消息
    ↓
构建记忆查询
    ↓
Active Memory Blocking Sub-Agent
    ├── 调用 memory_search / memory_get
    ├── 找到相关记忆 → 生成摘要（≤220 字符）
    │      ↓
    │   注入隐藏 system context（<active_memory_plugin> 标签）
    │      ↓
    │   主回复生成（Agent 看到了注入的记忆）
    │
    └── 没有相关记忆 → 返回 NONE
           ↓
       主回复生成（无额外上下文）
```

### 关键设计决策

1. **阻塞式**：在主回复前运行，不是异步
2. **隐藏注入**：用户看不到 `<active_memory_plugin>` 标签
3. **有限工具**：只能用 `memory_search` 和 `memory_get`
4. **超时控制**：硬超时（默认 15s），连接弱时返回 NONE
5. **临时 Session**：Sub-Agent 的 transcript 默认临时创建后立即删除

## 配置详解

### 查询模式（queryMode）

| 模式 | Sub-Agent 看到什么 | 延迟 | 适用场景 |
|------|------------------|------|---------|
| **message** | 仅最新用户消息 | 最低 | 稳定偏好召回，跟进不需上下文 |
| **recent**（推荐） | 最新消息 + 近几轮对话 | 中 | 平衡速度和对话接地 |
| **full** | 完整对话 | 最高 | 召回质量最重要 |

### 提示风格（promptStyle）

| 风格 | 说明 | 适用场景 |
|------|------|---------|
| **balanced** | 通用默认 | recent 模式默认 |
| **strict** | 最不积极，几乎不返回 | 减少噪音 |
| **contextual** | 最重视连续性 | full 模式默认 |
| **recall-heavy** | 更愿意在软匹配上返回 | 最大化召回 |
| **precision-heavy** | 除非明显匹配否则 NONE | 最大化精确 |
| **preference-only** | 只关注偏好/习惯/口味 | 个人偏好场景 |

### 模型回退链

```
显式 plugin model → 当前 session model → Agent 主模型 → 内置远程默认
```

### 使用门控（四重门控）

```
plugin enabled + agent id targeted + allowed chat type + eligible persistent session = 运行
```

任何一个不满足 → Active Memory 不运行。默认只在 **DM（直接消息）** 中运行。

### 不适用场景
- ❌ 无头一次性运行
- ❌ Heartbeat/后台任务
- ❌ 子 Agent/内部辅助
- ❌ 自动化工作流
- ❌ 隐式个性化会令人惊讶的场景

## 价值分析

### 解决的核心痛点

**之前**：
- 用户："帮我点外卖" → Agent 不知道偏好 → "你喜欢什么？"
- 用户必须说"搜索记忆，我之前说过喜欢柠檬辣翅"
- 记忆存在但不被主动使用 = 形同虚设

**之后**：
- 用户："帮我点外卖" → Active Memory 在后台搜索 → 找到"柠椒翅+蓝芝士"
- Agent 直接说："要不要老样子——柠椒翅配蓝芝士？"
- 记忆变成**主动的合作伙伴**而非被动的文件柜

### 量化收益

| 维度 | 之前 | 之后 |
|------|------|------|
| 用户重复解释 | 频繁 | 大幅减少 |
| 掉线的对话上下文 | 常见 | 自动接续 |
| 记忆搜索触发率 | 依赖用户主动 | 每次对话自动 |
| 个性化感知 | "这AI没记性" | "它记得我" |
| 延迟代价 | 0 | +1~15s（可调） |
