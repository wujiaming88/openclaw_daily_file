# Part 1：OpenClaw 记忆系统全景架构

## 设计哲学

> "模型只'记住'被保存到磁盘的东西——没有隐藏状态。"

OpenClaw 的记忆系统建立在一个极简但深刻的原则上：**一切记忆都是 Markdown 文件**。没有黑盒数据库，没有不可见的状态——人类随时可以打开文件查看和编辑。

## 三层记忆文件

| 文件 | 用途 | 加载时机 |
|------|------|---------|
| **MEMORY.md** | 长期记忆：持久事实、偏好、决策 | 每个 DM 会话开始 |
| **memory/YYYY-MM-DD.md** | 每日笔记：运行上下文和观察 | 自动加载今天+昨天 |
| **DREAMS.md** | 梦境日记（实验性）：dreaming 摘要 | 可选，人工审查用 |

位置：`~/.openclaw/workspace-<agent>/`

## 四层架构全景

```
┌─────────────────────────────────────────────────────┐
│              用户/Agent 交互层                        │
│  "记住我喜欢 TypeScript"  /  memory_search           │
└──────────────┬──────────────────────────────────────┘
               ↓
┌─────────────────────────────────────────────────────┐
│         Active Memory（主动记忆层）                   │  ← 新特性！
│  Blocking Memory Sub-Agent                           │
│  在主回复前主动搜索并注入相关记忆                      │
└──────────────┬──────────────────────────────────────┘
               ↓
┌─────────────────────────────────────────────────────┐
│     Active Memory Plugin（记忆运行时层）              │
│  memory-core（默认）/ QMD / Honcho                    │
│  ┌──────────┬──────────┬──────────┐                  │
│  │ Recall   │ Search   │ Dreaming │  ← 新特性！      │
│  │ 召回     │ 搜索     │ 梦境整理  │                  │
│  └──────────┴──────────┴──────────┘                  │
│  memory_search / memory_get 工具                      │
│  Compaction 前自动 Flush                              │
└──────────────┬──────────────────────────────────────┘
               ↓
┌─────────────────────────────────────────────────────┐
│         Memory Wiki（知识编译层）                      │  ← 新特性！
│  Claims / Evidence / Provenance / Dashboards          │
│  wiki_search / wiki_get / wiki_apply / wiki_lint      │
│  可选 Obsidian 集成                                   │
└──────────────┬──────────────────────────────────────┘
               ↓
┌─────────────────────────────────────────────────────┐
│           存储引擎层                                  │
│  SQLite + FTS5 (关键词)                               │
│  + Embeddings (向量: OpenAI/Gemini/Voyage/Mistral)    │
│  + Hybrid Search (混合搜索)                           │
│  sqlite-vec 可选加速                                  │
└─────────────────────────────────────────────────────┘
```

## 各组件的职责边界

| 组件 | 负责什么 | 不负责什么 |
|------|---------|-----------|
| **Active Memory** | 主动搜索、主回复前注入 | 存储、索引、晋升 |
| **memory-core** | 召回、搜索、索引、Dreaming、晋升 | 知识编译、溯源 |
| **Memory Wiki** | 知识编译、溯源、矛盾追踪、仪表板 | 召回、晋升、Dreaming |
| **存储引擎** | 关键词/向量/混合搜索 | 语义理解、决策 |

## 记忆后端选择

| 后端 | 特点 | 适用场景 |
|------|------|---------|
| **Builtin (SQLite)** | 开箱即用，FTS5+向量+混合搜索 | 默认推荐 |
| **QMD** | 本地 sidecar，reranking，查询扩展 | 进阶用户 |
| **Honcho** | AI 原生跨会话记忆，用户建模 | 多 Agent 感知 |

## Embedding Provider 自动检测

| Provider | 自动检测 | 默认模型 |
|----------|---------|---------|
| OpenAI | ✅ | text-embedding-3-small |
| Gemini | ✅ | 支持多模态(图片+音频) |
| Voyage | ✅ | — |
| Mistral | ✅ | — |
| Ollama | ❌ (需手动) | 本地 |
| Local | ✅ (首选) | GGUF, ~0.6GB |

## Compaction 前自动 Flush

在 Compaction（上下文压缩）之前，OpenClaw 自动运行一个静默轮次，提醒 Agent 把对话中的重要上下文保存到记忆文件。**默认开启，无需配置**。

这是防止上下文丢失的关键安全网。
