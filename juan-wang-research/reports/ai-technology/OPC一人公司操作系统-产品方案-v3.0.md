# OPC 数字员工平台：让每个创业者拥有自己的 AI 团队

> **一句话**：非技术背景的 OPC 创业者，注册即用，像招人一样招数字员工，像发微信一样指挥它们工作。

---

## 第一章：我们要解决什么问题

### 1.1 目标用户画像

我们的用户**不是**技术人员，而是：

| 画像 | 描述 |
|------|------|
| **张姐** | 38 岁，做跨境电商，之前在大厂做运营，去年出来单干，不会写代码 |
| **小陈** | 27 岁，做知识付费，擅长写内容和做课，但不懂 AI 和技术 |
| **李总** | 45 岁，做咨询顾问，有 20 年行业经验，电脑只会用 Office |
| **阿明** | 30 岁，独立开发者做 SaaS，技术很强但没精力做销售和客服 |

他们的共同点：**有业务能力，但没有技术背景（或没有精力处理技术）**，需要帮手但雇不起人或不想雇人。

### 1.2 他们今天怎么做的

```
现状：拼凑 5-10 个工具，手动当"中间人"

ChatGPT / 豆包 → 写文案、想方案
企业微信 / 飞书 → 和客户聊天
Excel / 飞书表格 → 记账、管客户
Canva / 稿定设计 → 做图
小红书 / 抖音后台 → 发内容
Zapier / Make → 自动化（大多数人不会用）

痛点：
❌ 每个工具都是独立的，信息不互通
❌ AI 不记得之前聊了什么（每次要重新说背景）
❌ 不能自动做事（每天手动给 AI 下达重复指令）
❌ 没人帮我盯着（客户消息半天没回、库存告急没人提醒）
❌ 要学各种工具，学习成本高
```

### 1.3 我们提供什么

```
一句话：打开平台 → 招一个数字员工 → 用飞书/钉钉指挥它工作

用户不需要知道:
  ❌ 什么是 Agent、LLM、Prompt
  ❌ 什么是 API、JSON、Docker
  ❌ 什么是 OpenClaw、Gateway、Workspace

用户只需要知道:
  ✅ 我要招一个"销售助理"
  ✅ 告诉它我的产品和价格
  ✅ 在飞书上@它就能指挥它干活
  ✅ 它会自己每天给我发日报
```

## 第三章：Harness Engineering 如何保障体验

用户感受到的是"零门槛"和"好用"，但底层需要精密的工程保障。这是 Harness Engineering 在我们平台中的落地方式：

### 3.1 为什么需要 Harness Engineering

```
没有 Harness Engineering:
  → 数字员工忘记产品信息（记忆丢失）
  → 数字员工乱说价格（没有约束）
  → 数字员工自作主张给客户承诺（权限失控）
  → 数字员工越来越慢（上下文爆炸）
  → 数字员工之间信息不通（孤岛）

有 Harness Engineering:
  → 5 层记忆保证"教过的它一定记得"
  → 知识库兜底保证"价格等关键信息永远准确"
  → Tool Policy 保证"该确认的一定确认"
  → LCM 压缩保证"用越久反而越快"
  → Agent 间通信保证"团队内信息共享"
```

### 3.2 六大工程保障 → 六个用户承诺

| 用户承诺 | Harness 实现 | 对应 OpenClaw 能力 |
|---------|-------------|------------------|
| **"教一次就记住"** | 多层记忆系统 | MEMORY.md + LCM + Memory Search + shared/knowledge |
| **"关键信息不出错"** | 知识库驻留 + 渐进加载 | Workspace 文件注入 + Skills 按需加载 |
| **"重要的事一定问我"** | 系统级护栏 | Tool Policy + Exec Approvals + 审批规则 |
| **"用越久越好用"** | 持续学习 + 上下文管理 | Memory Flush + LCM Compaction + 知识积累 |
| **"员工之间能配合"** | Agent 编排 | Sub-agents + sessions_send/spawn + 共享文件 |
| **"不用操心技术"** | 平台自动化 | Gateway 编排 + 自动配置 + 热更新 |

### 3.3 "教一次就记住"的工程实现

这是用户最核心的体验期望，也是 Harness Engineering 最核心的落地：

```
用户说: "我们的退货政策是 7 天无理由退货"
         │
         ▼
    平台判断: 这是一条业务知识
         │
         ├─ 即时生效:  写入当前会话上下文
         │            数字员工立刻能用
         │
         ├─ 短期记忆:  写入 Memory（LCM 管理）
         │            本次对话结束后仍然记得
         │
         ├─ 长期记忆:  写入 MEMORY.md
         │            重启、压缩后仍然记得
         │
         └─ 结构化:   写入 shared/knowledge/policies.md
                     所有数字员工都能查到
                     创始人在管理台也能看到/编辑

检索时:
  客户问"能退货吗？" 
  → Memory Search 语义检索 → 命中"7天无理由退货"
  → 数字员工准确回答
```

### 3.4 "重要的事一定问我"的工程实现

这是创始人信任数字员工的基础：

```
用户在"招聘"时选择:
  ✅ 回复客户咨询 → 自动
  ✅ 跟进邮件 → 自动（事后汇报）
  ⚠️ 报价/折扣 → 需要我确认
  ⚠️ 退款承诺 → 需要我确认
  🚫 修改产品信息 → 禁止
  🚫 删除客户数据 → 禁止

         │
         ▼ 平台自动生成

Tool Policy:
  allow: [web_search, message:read, message:send]
  deny: [exec, write:knowledge, delete]

Approval Rules:
  if action contains "报价|折扣|优惠":
    require_human_confirmation = true
  if action contains "退款|赔偿":
    require_human_confirmation = true

SOUL.md 注入:
  "你在以下情况必须先征求创始人确认，不能自作主张：
   1. 给客户任何折扣或优惠
   2. 承诺退款或赔偿
   3. 承诺功能或交付日期"

三层保障:
  Layer 1: SOUL.md prompt 约束（模型遵循）
  Layer 2: Tool Policy 硬拦截（系统强制）
  Layer 3: 审批推送到 IM（人类确认）
```

### 3.5 "员工之间能配合"的工程实现

```
创始人: "帮我查查上个月的销售情况，然后写一篇复盘"

主管数字员工（CEO Agent）接收到指令
  │
  ├─ 拆解任务 1: 查销售数据
  │  → sessions_spawn → 销售助理
  │  → 销售助理查询记忆和数据 → 返回结果
  │
  ├─ 拆解任务 2: 写复盘文章
  │  → 将销售数据传给内容运营
  │  → sessions_send → 内容运营
  │  → 内容运营撰写复盘 → 返回草稿
  │
  └─ 汇总 → 推送给创始人:
     "📊 上月销售复盘
      数据：GMV ¥185K，环比 +12%，新客 23 个
      [查看完整复盘文章]"

用户感受: 我说了一句话，团队就协作完成了
底层实现: Sub-agents + sessions_send + 共享文件
```

## 第四章：行业解决方案

### 4.1 按行业预配置的数字团队

用户选完行业后，平台推荐**开箱即用的数字员工组合**：

| 行业 | 推荐团队 | 开箱即用能力 |
|------|---------|------------|
| **跨境电商** | 客服(多语言) + 运营 + 选品分析 | 自动回复客户、Listing 优化建议、竞品监控日报 |
| **SaaS 产品** | 客服 + 销售跟进 + 内容运营 | FAQ 自动回答、线索跟进提醒、产品博客生成 |
| **知识付费** | 内容创作 + 社交运营 + 客服 | 课程文案、小红书/公众号内容、学员答疑 |
| **咨询顾问** | 研究分析 + 客户管理 + 文档 | 行业报告、客户跟进、方案文档生成 |
| **跨境贸易** | 客服(多语言) + 采购 + 物流跟踪 | 多语言客户沟通、供应商比价、物流状态追踪 |
| **自由开发者** | 项目管理 + 客户沟通 + 测试 | 需求整理、客户进度同步、Bug 报告 |

### 6.3 六周计划

#### Phase 1（Week 1-2）：Harness 生成引擎 + 招聘流程

这是 MVP 最核心的部分——**把用户的自然语言意图，转化为 OpenClaw 的 Harness 配置**。

| 任务 | 说明 |
|------|------|
| **Harness 模板库** | 为 3 个岗位（客服/销售/内容运营）编写高质量的 AGENTS.md、SOUL.md、Tool Policy 模板 |
| **意图→Harness 生成器** | 用户的对话内容（产品名、价格、FAQ）+ 模板 → 个性化的 Harness 文件 |
| **对话式招聘流程** | Web 页面上的引导式对话（不是表单），收集用户信息 |
| **员工管理 API** | 创建/配置/启停 Agent → 操作 Gateway 配置 + workspace 文件 |
| **员工列表 UI** | "我的数字团队"页面 |

**Harness 生成器核心逻辑**：

```
输入:
  - 岗位模板: "客服助理"
  - 用户信息: { 产品名: "超级选品", 价格: "月299/年2399", FAQ: [...] }
  - 权限选择: { 自动回复: true, 报价确认: true, 退款禁止: true }

输出:
  - AGENTS.md:  基于模板 + 用户信息 渲染
  - SOUL.md:    基于性格选项 渲染
  - Tool Policy: 基于权限选择 生成
  - shared/knowledge/product.md: 基于产品信息 生成
  - Cron Jobs:  基于岗位预置（客服→每日客服汇总 Cron）
  
方法:
  - 不是字符串拼接——用 LLM 根据模板和用户信息生成高质量 Harness
  - 模板是 80 分的底线，LLM 个性化到 95 分
  - 关键字段（价格、政策）从结构化数据注入，不依赖 LLM 生成
```
