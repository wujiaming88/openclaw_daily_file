# Part 3：上下文管理与压缩 + 权限安全模型

## 三层上下文压缩

Claude Code 最精密的工程之一——为长会话设计的三层压缩策略：

### Layer 1: MicroCompact（零 API 成本）
- **触发**：工具输出过旧时
- **方式**：本地直接裁剪缓存内容，不调用 API
- **特点**：快速、免费、透明
- **场景**：旧的工具输出被直接缩减

### Layer 2: AutoCompact（接近上限时）
- **触发**：对话 token 接近 98% 上下文窗口上限
- **方式**：
  1. 预留 13,000 token 缓冲
  2. 生成最多 20,000 token 的结构化摘要
  3. 内置断路器——连续 3 次压缩失败后停止重试
- **特点**：自动触发，无需人工干预
- **关键元数据保留**：图片和 PDF 被剥离

### Layer 3: Full Compact（完整重压缩）
- **触发**：手动 `/compact` 或极端情况
- **方式**：
  1. 压缩整个对话
  2. 重新注入最近访问的文件（每文件上限 5,000 token）
  3. 注入活跃的计划（plans）和相关技能 schema
  4. 压缩后工作预算重置为 50,000 token
- **特点**：最彻底，但代价最高

### 压缩的陷阱

> "压缩可能丢失重要细节。实际解决方案：把所有关键内容放在 CLAUDE.md 里——每轮都会重新读取。"

Anthropic 的研究发现：对于超长会话，**完整上下文重置**（新 Agent 实例从交接产物接手）有时比压缩效果更好——编排更复杂，但上下文保真度更高。

## 输出截断与 Token 管理

| 配置 | 默认值 | 说明 |
|------|--------|------|
| MCP 工具输出上限 | 25,000 token | 超过时警告 |
| 警告阈值 | 10,000 token | 提示输出偏大 |
| 大结果模式 | 500,000 字符 | 持久化到磁盘而非保留在上下文 |

## 会话管理

- **会话存储**：`~/.claude/projects/` 下的 JSONL 文件
- **每条消息、工具调用、结果**都被保存
- **文件快照**：修改前自动快照，支持回退（checkpoint）
- **会话独立**：每个新会话是全新上下文窗口
- **恢复与分叉**：`claude --continue` / `claude --resume`

## 记忆系统

Claude Code 有两个互补的记忆机制：

### CLAUDE.md（用户编写）
- **位置层级**：
  - Managed Policy（组织级）→ `/Library/Application Support/ClaudeCode/CLAUDE.md`
  - 项目级 → `./CLAUDE.md` 或 `./.claude/CLAUDE.md`
  - 用户级 → `~/.claude/CLAUDE.md`
  - 本地级 → `./CLAUDE.local.md`
- **最佳实践**：< 200 行，具体可验证的指令
- **支持 @import 语法**：`@path/to/file` 导入其他文件
- **每次会话开始时加载**

### Auto Memory（Claude 自动记录）
- Claude 自动记录学习到的模式、偏好、命令
- 保存在 MEMORY.md 中
- **加载限制**：前 200 行或 25KB（先到为准）
- 子 Agent 可以有自己的 Auto Memory

## 权限与安全模型

### Deny-First 规则管道

每个工具调用都经过权限检查管道：

```
工具调用 → Deny 规则检查 → Ask 规则检查 → Allow 规则检查
           ↓                ↓                ↓
         直接拒绝          提示用户确认       静默放行
```

**Deny 永远优先** —— 被拒绝的模型无法通过说服绕过。

### 三级权限模型

| 级别 | 操作类型 | 处理方式 |
|------|---------|---------|
| Tier 1: Auto-approved | 只读操作（文件读取、搜索、代码导航） | 静默放行 |
| Tier 2: Prompt | 受控的状态修改（文件编辑、特定 shell 命令） | 提示确认或 Auto Mode 分类 |
| Tier 3: Block/Require Approval | 高风险操作（系统修改、工作目录外操作、数据外泄模式） | 必须人工审批 |

### Auto Mode 的安全分类器

2026 年 3 月发布的 Auto Mode 引入了背景分类器：

- **运行模型**：Sonnet 4.6（独立实例）
- **输入**：用户请求 + 工具调用
- **刻意不看**：模型的散文输出 —— 防止模型用修辞"说服"分类器放行
- **决策**：自动放行 or 仍然提示用户

### 六种权限模式（官方）

| 模式 | 说明 |
|------|------|
| default | 标准权限检查 |
| acceptEdits | 自动接受编辑 |
| plan | 只读计划模式 |
| auto | Sonnet 分类器自动判断 |
| dontAsk | 不提问（宽松） |
| bypassPermissions | 绕过权限（最宽松） |
