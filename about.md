# 关于卷王小组

卷王小组是一个专注于深度学习和高效研究的多代理AI团队，由小帅于2026年3月1日创建。

## 团队定位

卷王小组致力于**卷出高度，卷出质量！** 我们专注于：

- **深度研究**: 深入分析技术架构和设计理念
- **高效协作**: 多代理协同工作，发挥各自专长
- **知识沉淀**: 系统化整理和保存研究成果
- **质量保证**: 严格的质量检查和评审机制

## 团队架构

卷王小组采用**5人协作模式**，每个角色都有明确的职责和专长：

### 团队角色

| 角色 | 模型 | 专长 | 职责 |
|------|------|------|------|
| **团队协调员** | seed-2-0-mini | 任务调度、状态管理 | 路由任务、跟踪状态、优先级决策、最终交付 |
| **研究员** | glm-4-7 | 资料搜集 | 搜集资料、搜索论文、信息挖掘 |
| **分析师** | deepseek-v3-2 | 深度分析 | 深度分析、要点总结、逻辑梳理 |
| **综合者** | glm-4-7 | 报告撰写 | 整合成果、撰写报告、知识沉淀 |
| **评审员** | glm-4-7 | 质量检查 | 质量检查、准确性验证、建议改进 |

### 工作流程

```
研究任务流程:
  Orchestrator → Researcher → Analyst → Synthesizer → Reviewer → Orchestrator
  (接收任务)    (搜集资料)    (深度分析)    (整合报告)    (质量检查)    (最终交付)
```

## 核心能力

### 1. 多源资料搜集
- 支持ArXiv学术论文搜索
- 支持多种搜索引擎（web-search-plus、xai-search、tavily-search）
- 自动整理和引用管理

### 2. 深度技术分析
- 代码任务使用DeepSeek模型（代码专长）
- 复杂推理使用专业版模型
- 逻辑梳理和关键发现提取

### 3. 结构化报告撰写
- 自动整合多源研究成果
- 生成结构化、编号章节的报告
- 知识总结和沉淀

### 4. 严格质量保证
- 多维度质量检查
- 准确性验证
- 改进建议和最佳实践

## 研究成果管理

### 输出规范
- **优先单一文件**: 最终报告最好是一个文件，确保对读者阅读优化
- **目录结构**: 使用清晰的目录结构
- **文档化**: 包含目录、内容表格、编号章节

### Git集成
- **自动推送**: 综合和评审完成后自动推送到远端仓库
- **提交格式**: `卷王小组研究成果: {任务标题} - {日期}`
- **仓库地址**: git@github.com:wujiaming88/openclaw_daily_file.git
